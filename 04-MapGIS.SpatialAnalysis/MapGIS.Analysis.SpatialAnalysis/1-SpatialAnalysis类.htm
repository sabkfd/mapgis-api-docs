MultiBuffer(

   SFeatureCls sfcls,

   SFeatureCls dsfcls,

   List<double> radlst,

   ObjectIDs oids,

   BufJointType type,

   bool isDissolve,

   bool isDynPrj

)

bool

执行成功返回true，失败返回false

多重缓冲分析

说明：

【图形参数】

缓冲区分析结果中图形参数随机。

【属性规则】

只有默认属性字段（指MapGIS中区的默认属性字段），并自动更新。

【详细说明】

1.6x和7x处理方法相同；

2.无需设置容差（默认容差值未知）；

3.对于多级缓冲区分析，合并缓冲是将缓冲结果中同级缓冲结果区中相交的区合并为一个区；

4.对于源中的复合要素，不论isDissolve为true或false，复合元素内各个元素的属于同一级的缓冲结果若有重叠则会合并成一个区，没有重叠则不合并，各自成单区；

5.多级缓冲区分析中的投影半径用radlst记录，radlst中的每个值指代各级缓冲半径，及各级缓冲中到源实体的距离，非两级缓冲之间的距离差；

6.线的多级缓冲只能进行恒定半径的双边等半径缓冲。

sfcls：待进行缓冲区分析的简单要素类

dsfcls：结果简单要素类，一般是区类型的简单要素类，该结果简单要素类需创建或指定

radlst：多重缓冲的缓冲半径列表

oids：对简单要素类进行局部缓冲区分析的要素ID序列，给定NULL值代表整个执行缓冲区分析

type：Buffer拐角类型,可设置圆头和平头两种类型

isDissolve：是否缓冲区合并

isDynPrj：是否为动态投影

剪断线引用方式说明：

1、NEW_FEATURE，即生成新的独立要素，例如某个线类型的简单要素类中，剪断操作之前有五个独立线对象（或称线要素），假设这五个线对线均被剪断且仅剪断一次，由于采用NEW_FEATURE引用方式，该简单要素类中将多出五个独立线对象，多出的五个线对象继承了原所在线对象的属性信息，共计十个独立线对象（浏览该简单要素类属性列表，即可发现有十条记录）。

2、UPDATE_FEATURE，即更新原有要素引用信息，用户在未设置情况下，系统默认采用此种方式，不会改变原有线对象数目。

mode：剪断线类型（或称剪断线引用方式），其值是枚举类型ClipRefMode（NEW_FEATURE-生成新的独立要素、UPDATE_FEATURE-更新原有要素引用信息[默认情况]）

SP_Buffer(

   SFeatureCls srcSfcls,

   SFeatureCls desSfcls,

   SPBufferOption option,

   ObjectIDs oids

)

bool

执行成功返回true，失败返回false

缓冲区分析

说明：

【属性】

1.不合并的缓冲："系统默认字段" + "用户字段"

2.合并的缓冲："系统默认字段"

【图形参数】

1.由SP_BUFFER_PARAM参数中的nClr决定；

【详细说明】

1.融合：若IsDissolve为True，则将缓冲结果中相交的区融合为一个区(只控制几何对象之间是否进行融合)。

2.复合要素：

1)对于源中的复合要素，复合元素内各个元素的缓冲结果若有重叠则融合为一个区，没有重叠则不融合，各自成单区。与IsDissolve无关。

2)当处理为复合要素，即MultiFeatureOpr=1时（只在IsDissolve=True的前提下有作用）,则将缓冲得到的结进行部融合，并将融合的结果组合为一个复合要素输出。

3.动态投影：

1)使用场景：对经纬度坐标下的数据，按线性单位（米）进行缓冲区分析。

2)参数控制：当IsDynPrj=true且DynPrjDis缓冲距离值有效时，则采用动态投影的方式进行缓冲分析。

3)误差说明：对于小范围数据（3度带内），结果误差较小。对于跨带数据，结果误差较大。

4)一点建议：对于大范围的跨带经纬度数据，建议用户根据具体业务选择合适的投影先对数据进行投影变换，再进行缓冲。

srcSfcls: 源简单要素类

desSfcls: 目的简单要素类

option: 缓冲参数

oids: OID列表（不为空则只对该OID列表图元进行缓冲

【属性规则】

SP_Dissolve(

   SFeatureCls srcSfcls,

   SFeatureCls desSfcls,

   SPDissolveOption option,

   ObjectIDs oids

)

bool

执行成功返回true，失败返回false

融合区（合并区）

说明：

【属性规则】

1.属性结构：系统默认字段 + 用户字段

2.属性记录：结果图元的属性记录来自对应的原始融合图元集合中的第一个图元的属性。

【图形参数】

1.由SP_DISSOLVE_PARAM参数中的nClr决定；

【详细说明】

1.接口命名：根据算法输出效果，同时参考ArcGIS和SuperMap对应的功能命名，最终决定新算法命名为Dissolve（融合区）更合理。老算法命名为Merge(合并区)。

2.融合后的几何形态：

1)相离几何：结果几何形态保持不变。

2)相邻几何：结果几何融合为一个图元。

3)重叠几何：结果几何融合为一个图元。

3.应用场景：

根据"局部融合/全局融合"和"按属性融合/不按属性融合",可组合产生4种使用场景：

1)局部按属性融合：将OID列表指定的图元按属性字段值，划分为几个"融合图元集合"（一堆图元），然后对这几个集合分别进行融合。

多环缓冲区（多级缓冲区）

说明：

【属性】

1.不合并的缓冲："系统默认字段" + "用户字段"

2. 合并的缓冲："系统默认字段"

【图形参数】

1.由SP_MURING_BUFFER_PARAM参数中的nClr决定；

【详细说明】

1.融合：

1.1)若IsDissolve为True，则将同一级的缓冲结果中相交的区融合为一个区(只控制几何对象之间是否进行融合)。

1.2)若IsDissolve为False，对图元逐一按缓冲半径由小到大进行缓冲，第一级生成最小缓冲区，后续缓冲区将是围绕最小缓冲区的环形。（不对重叠图元进行合并）

2.复合要素：

2.1)对于源中的复合要素，复合元素内各个元素的缓冲结果若有重叠则融合为一个区，没有重叠则不融合，各自成单区。与IsDissolve无关。

2.2)当处理为复合要素，即MultiFeatureOpr=1时（只在IsDissolve=True的前提下有作用）,则将同一级缓冲得到的结进行融合，并将融合的结果组合为一个复合要素输出。

3.缓冲半径列表：

3.1)含义：缓冲半径用distances记录，distances中的每个值代表各级的缓冲半径。

3.2)注意：distances的值并非两级缓冲之间的距离差！！！

Buffer 方法示例

C#

 
using MapGIS.GeoDataBase;
using MapGIS.GeoObjects;
using MapGIS.Analysis.SpatialAnalysis;
using MapGIS.GeoObjects.Geometry;
 
//地理数据库服务器对象
Server srv = null;
//地理数据库对象
DataBase db = null;
 
srv = new Server();
//连接数据源
srv.Connect("mapgislocal", "", "");
//打开数据库
db = srv.OpenGDB("示例数据");
 
//待缓冲区分析简单要素类
SFeatureCls sfcls = null;
//结果简单要素类
SFeatureCls destSfcls = null;
 
//打开分析对象
sfcls = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls.Open("省会城市", 1);
//创建结果要素类
destSfcls = db.GetXClass(XClsType.SFCls) as SFeatureCls;
destSfcls.Create("buffer", GeomType.Reg, 0, 0, null);
 
//创建缓冲区分析分析对象
SpatialAnalysis sa = new SpatialAnalysis();
//设置容差
sa.Tolerance = 0.0001;
//设置缓冲区分析分析参数
BufferOption option = new BufferOption();
//设置左右半径
option.LeftRad = 10;
option.RightRad = 10;
option.AngleType = BufJointType.Circle_Angle;
option.IsDissolve = false;
//设置待缓冲区分析对象
ObjectID objId = new ObjectID();
objId.Int64Val = 12;
ObjectIDs objIds = new ObjectIDs();
objIds.Append(objId);
//执行分析
sa.Buffer(sfcls, destSfcls, objIds, option);
 
//关闭地理要素及数据库，断开服务连接
sfcls.Close();
destSfcls.Close();
db.Close();
srv.DisConnect();

//执行叠加分析
spa.OverLay(sfcls0, sfcls1, destSfcls, option);
 
//关闭地理要素及数据库，断开服务连接
sfcls1.Close();
destSfcls.Close();
db.Close();
srv.DisConnect();

备注

关于叠加分析参数option注意：
（1）须对此参数设置叠加分析类型属性，如“option.OverlayType = MapGIS.Analysis.SpatialAnalysis.OverlayType.Ovly_Inter;”即是相交分析；MapGIS.Analysis.SpatialAnalysis.OverlayType枚举了叠加操作的各种类型，如Ovly_Union(求并)，Ovly_Inter(相交)，Ovly_Sub(相减)，Ovly_SymDiff(对称差)，Ovly_Ident(判别)；而不同类型的简单要素类进行叠加可以进行的叠加操作有所不同，如下表所示：

线对区

相交、相减、求并、判别

区对区

相交、相减、求并、判别、对称差

（2）当简单要素类sfcls0,sfcls1具有严格正确的拓扑信息时, 设置option=null[默认情况]；当简单要素类sfcls0,sfcls1的拓扑信息不完整时，设置MapGIS.Analysis.SpatialAnalysis.OverlayOption.LabelPnt=1[采用label点方法]。

（3）容差半径需要在空间分析中的Tolerance属性字段中设置；

 

SpaGeoSql 方法示例

此示例用区类型简单要素类（regSfcls0）去查点类型简单要素（pntSfcls1），查询出包含在区要素中的点要素并生成新的点类型的简单要素类。

C#

 
using MapGIS.GeoDataBase;
using MapGIS.Analysis.SpatialAnalysis;
 
//定义变量
Server srv;
DataBase db;
SFeatureCls sfcls0 = null;
SFeatureCls sfcls1 = null;
SFeatureCls destSfcls = null;
SpatialAnalysis spa;
 
//变量初始化
srv = new Server();
//连接数据源
srv.Connect("mapgislocal", "", "");
//打开数据库
db = srv.OpenGDB("中国地图");
 
//打开待查询的源简单要素类0
sfcls0 = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls0.Open("regSfcls0", 1);
//打开待查询的源简单要素类1
sfcls1 = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls1.Open("pntSfcls1", 1);
//创建结果简单要素类
destSfcls = db.GetXClass(XClsType.SFCls) as SFeatureCls;
destSfcls.Create("SpaGeosql4", sfcls1.GeomType, 0, 0, null);
//初始化空间查询对象
spa = new SpatialAnalysis();
//包含关系
spa.SpaGeoSql(sfcls0, sfcls1, destSfcls, GSQLOperType.Within, false);
 
//关闭地理要素及数据库，断开服务连接
sfcls0.Close();
sfcls1.Close();
destSfcls.Close();
db.Close();
srv.DisConnect();

备注

（1）空间几何查询，其中查询/判别类型分为：EQUAL-相等：DISJOINT-相离；INTERSECT-相交；TOUCH-相邻；CROSS-相穿：WITHIN-包含于；CONTAIN-包含；OVERLAY-相叠。
（2）裁剪框简单要素类要求为区类型。
（3）容差半径需要在空间分析中的Tolerance 属性中设置。
