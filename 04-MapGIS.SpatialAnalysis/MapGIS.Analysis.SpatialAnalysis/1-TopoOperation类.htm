示例：请参考LinkLines( SFeatureCls, String, ObjectIDs, SFeatureCls, Double, Boolean, LinkLinesunionflag) 方法示例

说明：

【图形参数】

合并结果线图形参数与它的某个合并源线相同（合并源线：合并成该线的源线）。

【属性规则】

1.不按属性合并，且dsfcls不为null时，合并结果只保留默认属性（不论有没有线进行了合并），其他情况下均会保留非默认属性；

2.合并结果线的默认属性（指MAPGIS中线的默认属性值）值自动更新；

3.合并结果线的非默认属性值（若有）与它的某个合并源线相同且图形参数也与此源线相同。

【详细说明】

1.6x和7x处理方法相同；

2.根据属性合并值，合并的对象是属性值相等且相接的线，此处的属性值相等指的是属性值不为NULL且值相等，包括属性值等于空字符串，但串的长度相等的情况；

3.对7x数据，LinkLinesunionflag为MultiLine时要合并的线只是组合成多线，LinkLinesunionflag为notUnion时要合并的线会连接成一条单线；对6x数据。LinkLinesunionflag为MultiLine或anyLine都会连接成一条单线；

4.合并结果中源多线会被打散为单线（不论LinkLinesunionflag的值是多少）；对于多线自身，若其中的两单线相接，则一定会合并，合并结果是单线还是多线依旧上面anyLine条判断；

5.合并的最基本要求是两线邻接，若有三条线或跟多线共邻接于一个点，则这几条线都不会合并。

SearchSubReg(

   SFeatureCls sfcls,

   ObjectIDs oids

)

bool

执行成功返回true，失败返回false

简单要素类搜子区

说明：

【图形参数】不变，对于父区和子区都保持原来的图形参数。

【属性规则】会更新默认属性(面积/长度)。

【详细说明】

1.由Tolerance设置容差，建议设置为0.0001

2.不支持多区

3.当oids为null全局搜子区时，对所有区进行搜子区。若父区中有子区，子区中有孙区的情况，搜区结果就成了父区被子区挖掉变成带洞区，子区被孙区挖掉形变成带洞区，孙区不变；

4.当oids不为null部分搜子区时，逐一将传入的区作为父区，去找该区的真正子区，找到后就用子区去挖父区，父区就变成带洞区。

5.传入的区要保证没有拓扑错误，函数不做区拓扑错误检查，当有拓扑错误时，搜区结果不确定，可能不对。

sfcls：待处理区简单要素类

oids：待搜子区的ID序列列表，null表示全局搜子区

TopoCheck(

   SFeatureCls sfcls,

   int maxErr,

   RegTopoCheckOption opt,

   out List<RegTopoErrorInfo> errInfo

)

bool

执行成功返回true，查询出拓扑错误信息，失败返回false

重载. 拓扑查错 

说明：

【图形参数】不涉及图形参数操作

【属性规则】不涉及属性操作

【详细说明】

1.由Tolerance设置容差，建议设置为0.0001

2.多区的碎小区错误检查是对多区中每个单区的检查，若有一个单区为碎小区，则该多区由碎小区错误；

3.区拓扑查错依次进行3个方面的拓扑查错：①单个区错误查询；②碎小区查询；③重叠区查询，拓扑查错的顺序遵循两个规则：

①一旦一个区在错误检查中被标记，则该区与其他区的错误不再被标记。如：区A，B，C，三者id由小到大，A与B重叠，B与C重叠；则对A检查时检查到A与B重叠，A、B都被判定为错误区，不再算作拓扑查错的候选区，B，C间的重叠不会查到。

②一个区只要发现有错误，就不再进行后面类型的错误检查，因此有多种或多个同类型错误的区只能检查出一个错误。如，区A，B两者有重叠且A自相交，则对A的拓扑查错在检查到A的自相交错误后就将A判定为错误区，不再继续查错；又如A是一个多区，由A1，A2，A3三个部分组成，且A1，A2为碎小区，A3另一个区B有重叠，对A的拓扑检查只会查到有一个碎小区错误。

4.K9中单个区错误查询中的检查的顺序于与此不同，因此若一个区有多种类型的错误，在K9和K10中检查后显示的错误可能不同。K9的单个区错误查询中，狭窄区查错在自相交查错前，K10中则是先进行自相交查错再进行狭窄区查错。

sfcls：需要拓扑查错的简单要素类对象

maxErr：最多错误个数

opt：拓扑错误类型

errInfo：【输出】返回拓扑错误信息

TopoCheck(

   SFeatureCls sfcls,

   Rect rect,

   int maxErr,

   TopoCheckOption opt,

   out List<TopoErrorInfo> errInfo

)

bool

执行成功返回true，查询出拓扑错误信息，失败返回false

重载. 拓扑查错 

说明：

【图形参数】图形参数不变

【属性规则】属性不变

【详细说明】

1.只支持线简单要素类检查。不支持其他类型。

2.线简单要素类进行拓扑检查时会删除线上的重复点。

3.类成员MinLength必须大于0。

4.类成员Tolerance必须大于0。

5.支持对多线的检查，并且把多线作为一个整体进行检查，不会把多线打散。错误id号使用该多线的id号。举例说明：现有一个id号为10的多线，该多线包含两个CAnyLine类型的线，其中一条线存在自相交，那么拓扑检查结果，错误弧段号为10，错误类型为自相交。

6.线拓扑错误优先级含义说明：线拓扑错误有优先级的区分，对于像弧段自相交、弧段相交的拓扑错误，改正后又可能引发新的错误，所以要优先处理。而像微短线、坐标点重复等情况，处理后不会引发别的错误，优先级较低。总体来说，优先级由高到低依次为：孤立节点、悬挂弧段、弧段自相交、弧段相交、坐标点重复、重复弧段、微短弧段。

7.由于线拓扑错误有优先级的区分，当同时存在多种错误时，只显示优先级比较高的错误信息，所以无法一次全部查找完。同时当改了一个错误可能会引发另外一个错误。也会造成无法一次查找完的情况。

8.进行线的拓扑检查是为了后面的成区做准备。

rect：拓扑查错的逻辑范围

maxErr：最多错误个数

opt：拓扑错误类型

errInfo：【输出】返回拓扑错误信息

示例

CombineLabelCls 方法示例

C#

 
using MapGIS.GeoDataBase;
using MapGIS.Analysis.SpatialAnalysis;
 
//定义变量
Server srv;
DataBase db;
SFeatureCls sfcls0 = null;
SFeatureCls sfcls1 = null;
TopoOperation topoOper;
 
//变量初始化
srv = new Server();
//连接数据源
srv.Connect("mapgislocal", "", "");
//打开数据库
db = srv.OpenGDB("中国地图");
 
//打开待合并Label点简单要素类
sfcls0 = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls0.Open("lblPntSfcls", 1);
//打开合并到的区简单要素类
sfcls1 = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls1.Open("lblRegSfcls", 1);
 
//将Label点简单要素类对象合并到多边形简单要素类
topoOper = new TopoOperation();
bool brtn = topoOper.CombineLabelCls(sfcls0, sfcls1, true, null);
 
//关闭地理要素及数据库，断开服务连接
sfcls0.Close();
sfcls1.Close();
db.Close();
srv.DisConnect();

备注：进行Label归并操作时，请确保参与此操作的简单要素类处于未使用状态（或关闭状态）。

LinkLines(SFeatureCls,String,ObjectIDs,SFeatureCls,Double,Boolean,LinkLinesunionflag) 方法示例

指定属性字段(如"ID"),连接处的距离在给定容差半径范围之内,对ID属性值相等的两两线要素之间进行自动连接操作.

C#

 
using MapGIS.GeoDataBase;
using MapGIS.Analysis.SpatialAnalysis;
 
//定义变量
Server srv;
DataBase db;
SFeatureCls sfcls0 = null;
SFeatureCls dsfcls = null;
TopoOperation topoOper;
 
//变量初始化
srv = new Server();
//连接数据源
srv.Connect("mapgislocal", "", "");
//打开数据库
db = srv.OpenGDB("示例数据");
 
//打开待处理线简单要素类
sfcls0 = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls0.Open("主要河流", 1);
 
//创建结果线简单要素类
dsfcls = db.GetXClass(XClsType.SFCls) as SFeatureCls;
dsfcls.Create("link", sfcls0.GeomType, 0, 0, null);
 
//根据属性自动连接线
topoOper = new TopoOperation();
topoOper.LinkLines(sfcls0, "ID", null, dsfcls, 5, true, LinkLinesunionflag.anyLine);
 
//关闭地理要素及数据库，断开服务连接
sfcls0.Close();
dsfcls.Close();
db.Close();
srv.DisConnect();

备注：

（1）指定属性字段的值相等的两条线才能自动连接；

（2）两条线要素待连接处之间的距离必须在容差（radiu）范围之内才会自动连接。

TopoRegion 方法示例

C#

 
using MapGIS.GeoDataBase;
using MapGIS.Analysis.SpatialAnalysis;
using MapGIS.GeoObjects.Geometry;
 
//定义变量
Server srv;
DataBase db;
SFeatureCls sfcls0 = null;
SFeatureCls dsfcls = null;
ObjectIDs unionRegIds;
ObjectID unionRegId;
ObjectIDs dRegIds = null;
 
//变量初始化
srv = new Server();
//连接数据源
srv.Connect("mapgislocal", "", "");
//打开数据库
db = srv.OpenGDB("示例数据");
 
//打开待处理的线简单要素类
sfcls0 = db.GetXClass(XClsType.SFCls) as SFeatureCls;
sfcls0.Open("经纬线", 0);
 
//创建结果区简单要素类
dsfcls = db.GetXClass(XClsType.SFCls) as SFeatureCls;
dsfcls.Create("topoReg", GeomType.Reg, 0, 0, null);
 
//待处理线要素ID列表
unionRegIds = new ObjectIDs();
unionRegId = new ObjectID();
unionRegId.Int64Val = 1;
unionRegIds.Append(unionRegId);
 
//拓扑成区
TopoOperation topoOper = new TopoOperation();
topoOper.TopoRegion(sfcls0, unionRegIds, dsfcls, out dRegIds);
 
//关闭地理要素及数据库，断开服务连接
sfcls0.Close();
dsfcls.Close();
db.Close();
srv.DisConnect();

备注：

（1）拓扑成区操作，建议一般是对没有拓扑错误的线简单要素类进行此操作，构建成区简单要素类；

（2）拓扑成区操作，线简单要素类中待处理的要素需要是一个闭合的多边形才可以转换成功。
